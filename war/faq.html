EPPIC - Frequently Asked Questions

General

What does EPPIC stand for?

Evolutionary Protein-Protein Interface Classifier.

What can I use EPPIC for?

EPPIC mainly aims at classifying the interfaces present in protein crystal lattices in order to determine whether they are biologically relevant or not. In addition to that it provides information of general use for a number of structural biology applications:

  • "Molecular Potato" - Surface representation of each chain color coded by sequence entropy
  • Number, identity distribution and coverage of putative homologs in the MSA used for entropy calculation
  • Number and distribution of core residues in interfaces
  • Symmetry features of interfaces: the crystallographic operators generating each interface are provided and are depicted in red if they are conducive to infinite assemblies. Also, one can compare the left- and right-hand side of the Details table of each interface to see if it is isologous or heterologous

Usage

What are the advanced input parameters for?

  • Identity soft cut-off: The soft sequence identity cut-off for BLAST hits to be used in the alignment. If there are not enough sequences at this cut-off then threshold is relaxed until the hard cut-off is reached.
  • Identity hard cut-off: The hard sequence identity cut-off for BLAST hits to be used in the alignment. If there are not enough sequences at soft cut-off then threshold is relaxed until this hard cut-off.
  • Max num of sequences: The maximum number of sequences to be taken from the BLAST hits.
  • Search Mode: The PDB sequence is first matched to its UniProt entry and that sequence then used for blasting. In Global mode the full length UniProt is used, in local only the matching PDB part of the UniProt sequence is used.

If I change the input parameters and submit any structure, why does it start calculating?

We have PDB-wide results precalculated for our users. We use only the default parameters for this calculation. If you use any parameters other than the default ones, it triggers a calculation which can take several minutes depending on the structure.

Can I access the results of a job in which I submitted my own PDB/mmCIF file later?

Every new job run on the server is assigned a unique job identifier, a long alpha numerical string that is only known by the user that submits it and that is very hard to guess. It is recommended that you give an email address while submitting so that you receive the URL with the job identifier in your inbox. Otherwise you will have to bookmark it or keep a record of it yourself. It is always possible to retrieve the job by using the URL http://www.eppic-web.org/ewui/#id/<my_job_id>. Whether the job is still running or already done, the URL will show its current status and automatically display the final results whenever it is finished. You can even share the URL with colleagues. The jobs will be stored in our servers for at least 1 month.

Can I directly access the precomputed results of a PDB code?

The PDB-wide precomputed results can be accessed directly by using the permanent URLs: http://www.eppic-web.org/ewui/#id/<PDB_code>

What does the UniProt version on the home page indicate?

The UniProt version (e.g. 2013_11) on the homepage denotes the UniProt version used to calculate the results of EPPIC. UniProt is used to blast for homologs of the input structure, which are then used to calculate entropy scores. These homologs can also slightly change from one version of UniProt to the next (monthly releases), so the results can vary over time. The UniProt version used for a calculation is also provided in the header of the Sequence Information panel in the results page.

What does the version below the EPPIC logo in results page indicate?

We always keep improving EPPIC with bug-fixes and added features. The version displayed below the EPPIC logo indicate the version of EPPIC used to calculate the results. To find out what has changed in various versions, please refer our Release Log page.

Where can I get the sequence entropy information?

Surface representation of each chain color coded by sequence entropy can be downloaded from the icon in the sequence information panel. This icon is adjacent to the homologs information icon. The downloaded file is in PSE format, which can be visualized with PyMol (www.pymol.org). To date we do not have an integrated viewer to visualize it in our server .

What is the 3D Viewer option in the navigation panel for?

EPPIC allows you to visualize the interfaces in 3D. By clicking on the thumbnails in the interface table one gets this interactive 3D view. 3D Viewer window allows you to change the type of viewer. The available options are: browser embedded Jmol viewer (no need for local viewer), a PDB file downloadable to a local molecular viewer, or PyMOL session file (.pse) to be opened in local PyMOL.

Method

How does EPPIC work?

EPPIC relies on sequence evolution and geometry to detect the biological character of interfaces. The classifier uses a simple geometric measure, number of core residues, and two evolutionary indicators based on the sequence entropy of homolog sequences. Both aim at detecting differential selection pressure between interface core and rim or rest of surface. For further information please refer to Duarte et al., BMC Bioinformatics 2012.

What are core-surface and core-rim scores?

Core-surface and core-rim scores are the ones which rely on evolution to predict the biological nature of an interface. They are calculated as follows:

  • core-rim score: Ratio of evolutionary signal (sequence entropy) of core residues (at 70% burial) versus rim residues
  • core-surface score: A score (z-score like) of sequence entropy of core residues (at 70% burial) versus random samples of other surface residues

The scores are translated to calls using their cut-off values. This implies that if the calculated score is less than it's cut-off value the call is XTAL or BIO otherwise.

What does nopred in interface table refer to?

The nopred (no prediction) call can occur when there is not enough information available for EPPIC to make a decision (usually not enough sequence data).

What do the core sizes in the interface table indicate?

Number of core residues (at 95% burial) in each interface, indicating how good the packing in the interface is.

What is the difference between core-geometry and core-evolutionary residues?

Core-geometry residues are core residues at 95% burial and core-evolutionary are the core residues at 70% burial. To calculate the geometry score, cores at 95% burial are used and to calculate evolutionary scores, cores at 70% burial are used.

How is the final call predicted?

Final call is the final consensus call from evolution and geometry calls.

Miscellaneous

How should I cite EPPIC?

The main paper describing the EPPIC method is Duarte JM, Srebniak A, Schaerer MA, Capitani G, BMC Bioinformatics 2012. As well as citing the paper it is important that one cites the EPPIC software version (top right below the logo in results page) and even more important the UniProt database version (monthly releases) used for the evolutionary analysis. Our predictions depend on the sequence homologs found by searching the UniProt database so with every release of the database the predictions can change. UniProt is growing very fast these days so a few months of UniProt releases between two runs can make a difference.
